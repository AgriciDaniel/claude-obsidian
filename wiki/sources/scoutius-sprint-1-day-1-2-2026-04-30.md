## Decisions Logged

- **Next 16 + Tailwind 4 over Next 15 + Tailwind 3.** Architecture v1.0 frozen on the older versions; deviation accepted because Next 16 is a minor release with the same App Router and public API. Future Phase 2/3 documents will reference the actual versions in tree at that time.
- **Helpers in `public.` instead of `auth.`** Architecture R3 said `auth.user_role()`; deferred to `public.user_role_slugs()` because Supabase reserves `auth` schema and `auth.role()` is a built-in. Intent of R3 (RLS reads JWT, exposes tenant + roles) preserved.
- **`DATABASE_URL` is the Session pooler, not Transaction pooler.** Drizzle migrate uses advisory locks; transaction-mode pooler does not preserve them. Session pooler at port 5432 is the right pick for both runtime and migrations on Free tier.
- **`DIRECT_URL` is now optional with fallback to `DATABASE_URL`.** Supabase Free direct host is IPv6-only and macOS `getaddrinfo` returns ENOTFOUND for AAAA-only entries. Pooler is dual-stack; direct connection becomes a Phase 2/3 nicety once IPv6 routing is confirmed.
- **Google OAuth: temporarily using Supabase's shared/built-in credentials, not own Google Cloud client.** Setup & Costs frozen on own client (better consent-screen branding "Scoutius" instead of "supabase.com"). Recorded as a project memory: revert before MVP launch (after Vladosik evaluates MVP feel). See memory `project_scoutius_oauth_switch.md`.
- **Branch protection on `main` deferred to Sprint 2** when CI checks exist for the rules to require.
- **Initial commit author** rewritten from default global `00Hidden <hiddenweb3@gmail.com>` to repo-local `Vladyslav Bielkin <dep-growth@tools.l-a-b-a.com>` via `git commit --amend --reset-author --no-edit`.

## Mistakes Caught and Fixed

- I initially recommended Supabase's **new "Publishable / Secret" API keys** (`sb_publishable_*` / `sb_secret_*`). User pushed back. On reflection: Setup & Costs explicitly names legacy `anon` + `service_role` JWT keys, `@supabase/ssr` documentation and tutorials still center on legacy, and helper Postgres functions consume JWT claims via the legacy `SUPABASE_JWT_SECRET`. Reversed the recommendation; legacy keys used. The "frozen architecture" principle did exactly what it was supposed to.
- Drizzle Kit auto-generated `CREATE SCHEMA "auth"` and `CREATE TABLE "auth"."users"`. Supabase manages both — would have failed migration on apply. Patched to `CREATE SCHEMA IF NOT EXISTS` and removed the `CREATE TABLE`. (Snapshot still includes `auth.users` for FK type-safety; clean future generates.)
- `drizzle-kit migrate` failed silently with a spinner. Replaced with custom `scripts/migrate.ts` to surface the underlying `getaddrinfo ENOTFOUND` from the IPv6-only direct host.
- RLS smoke initially "failed" — false negative from BigInt vs string comparison (postgres-js returns `count(*)` as string by default). Fixed by `Number(c) !== 0` comparison; second run PASSED.
